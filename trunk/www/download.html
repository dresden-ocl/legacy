Where to download

This project is hosted on SourceForge and all sources and binaries produced around the Dresden OCL Toolkit are available from the project's site on SourceForge.

What to download

You can choose between downloading the release version accessible through the "Files" submenu and obtaining a current snapshot of the source via anonymous CVS access. Information on how to access the CVS repository is given on the project's site on SourceForge. Additionally you can browse the CVS repository using a web-based CVS interface.

Please be informed that the latest release version is outdated and that we therefore strongly encourage you to make use of the anonymous CVS access instead. The OCL Toolkit 2.0 is not contained in the release version.

For the OCL Toolkit 2.0 check out ocl20 and its subdirectories, for the OCL Toolkit 1.3 check out src and its subdirectories (browse the CVS repository to understand these directions).

Prerequisites

OCL Toolkit 2.0

For compiling and running the OCL Toolkit 2.0 you need:

  • Java SDK 1.4 or 1.5 with JAVA_HOME being set correctly
  • Jakarta Ant for the build process. Version 1.6.2+ recommended.
  • Recommended Hardware is 256MB+ of memory and at least a 400 MHz CPU

OCL Toolkit 1.3

For compiling and running the older OCL Toolkit covering the OCL 1.3 specification you need:

  • Java SDK 1.3. While the Toolkit generally also runs under SDK 1.4 the build process will fail when using the Java SDK 1.4.
  • Jakarta Ant for the build process.
  • Recommended Hardware is 256MB+ of memory and at least a 400 MHz CPU
