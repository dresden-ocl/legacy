Where and What to Download

This project is hosted on SourceForge and all sources and binaries produced around the Dresden OCL2 Toolkit are available from the project's site on SourceForge.

Eclipse Update Site

The easiest way to download Dresden OCL2 for Eclipse is via the Eclipse update manager. The update site of Dresden OCL2 for Eclipse is located at
http://dresden-ocl.sourceforge.net/downloads/updatesite/

Further information about the installation and use of Dresden OCL2 for Eclipse is provided in the manual available via this link (opens as a PDF in a new window).

Binaries and Source Code

Alternatively, you can choose between downloading the release version accessible through the "Files" submenu and obtaining a current snapshot of the source via anonymous SVN access. Information on how to access the SVN repository is given on the project's site on SourceForge. Additionally you can browse the SVN repository using a web-based SVN interface.

Please be informed that the latest release version is outdated and that we therefore strongly encourage you to make use of the anonymous SVN access instead.

Attention!

The binaries in the CVS which is also available via the project's site on SourceForge are outdated and will not be updated anymore! Use the SVN instead.

Prerequisites

Dresden OCL2 for Eclipse

For compiling and running the OCL2 Toolkit for Eclipse you need:

  • Java SDK 1.5 (or later) with JAVA_HOME being set correctly
  • Eclispe SDK (3.5 or later) including the Eclipse Modeling Framework (EMF) (2.5 or later)
  • Jakarta Ant for the build process. Version 1.6.2+ recommended (Only if you plan to build the Toolkit from SVN sources).
  • Recommended Hardware is 512MB+ of memory and at least a 1 GHz CPU

Changes in Different Releases

To investigate the changes and bugfixes introduces with the different releases of Dresden OCL2 for Eclipse you may also read the changelog provided with Dresden OCL2 for Eclipse (available from SVN via this link).

Prerequisites (Outdated Versions)

Dresden OCL2 Toolkit

For compiling and running the OCL2 Toolkit you need:

  • Java SDK 1.5 with JAVA_HOME being set correctly
  • Jakarta Ant for the build process. Version 1.6.2+ recommended (Only if you plan to build the Toolkit from SVN sources).
  • Recommended Hardware is 256MB+ of memory and at least a 400 MHz CPU

Dresden OCL Toolkit

For compiling and running the older OCL Toolkit covering the OCL 1.3 specification you need:

  • Java SDK 1.3. While the toolkit generally also runs under SDK 1.4 the build process will fail when using the Java SDK 1.4.
  • Jakarta Ant for the build process (Only if you plan to build the toolkit from CVS sources).
  • Recommended Hardware is 256MB+ of memory and at least a 400 MHz CPU
