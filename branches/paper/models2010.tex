\begin{abstract}
  In recent years, OCL has been 
	applied to various modelling languages including meta-meta-models 
	like the Meta Object Facility (MOF) or Ecore, various 
	Domain Specific Languages (DSLs) and the Unified Modelling Language (UML). It
	is rather common to reuse the same OCL parsers for multiple different modelling 
	languages. Nevertheless, dynamic OCL semantics and thus, OCL interpretation is
	often realized in a domain or implementation language specific manner. 
	In this paper, we propose an architecture for an OCL infrastructure that
  hides meta-models, models and model instances behind well-defined
  interfaces. We present an OCL interpreter that can interpret OCL constraints 
  on various different model instantiations and thus, enables reuse of the same 
  dynamic OCL semantics implementation for all these instantiation types.
  The architecture is evaluated in three case studies
  defining and interpreting OCL constraints on multiple models and model
  instances.

\keywords{OCL, OCL Infrastructure, MDSD, Modelling, Constraint Interpretation, Variability, Adaptation.}
\end{abstract}
