Dresden OCL2 for Eclipse - Usage examples

How to use Dresden OCL2 for Eclipse

Here you can find a few usage examples to learn, how to work with Dresden OCL2 for Eclipse.

Tutorials

  • An introduction into Dresden OCL2 for Eclipse

    This tutorial written by Claas Wilke and Ronny Brandt explains how to install Dresden OCL2 for Eclipse, how to load models and model instances and how to parse OCL constraints.

  • How to use the OCL2 Interpreter

    This tutorial written by Claas Wilke explains how the OCL2 Interpreter of Dresden OCL2 for Eclipse can be used to interpret OCL2 constraints for model instances.

  • How to use the Java Code Generator Ocl2Java

    This tutorial written by Claas Wilke explains how the Java code generator Ocl2Java of Dresden OCL2 for Eclipse can be used to generate AspectJ code for OCL constraints.

  • Examples provided with Dresden OCL2 for Eclipse

    This PDF written by Claas Wilke provides an overview over all examples which are provided with Dresden OCL2 for Eclipse.

  • Pivot Model Adapter Generation

    This tutorial written by Michael Thiele explains shortly how the Adapter classes to adapt the Pivot Model of Dresden OCL2 for Eclipse to a new domain-specific language can be generated using the Adapter Generator of Dresden OCL2 for Eclipse.

Screencasts

  • Creating AspectJ code with the Java Code Generator Ocl2Java

    This screencast written by Claas wilke shortly explains how the Java code generator Ocl2Java of Dresden OCL2 for Eclipse can be used to generate AspectJ code for OCL constraints.

  • Adapting a domain-specific language with the Pivot Model

    This screencast written by Mathias Bräuer shows how a domain-specific model can be imported into Dresden OCL2 for Eclipse.
    Attention: Please note, that the screencast was not created with the last release of Dresden OCL2 for Eclipse. Thus, some menu and wizard names changed since the screencast's creation.

  • Generically modeling the OCL Standard Library

    This screencast written by Mathias Bräuer shortly explains how to the OCL standard library has been modeled using the Pivot Model of Dresden OCL2 for Eclipse.
