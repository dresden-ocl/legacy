OCL Suite for Java
If you want to try OCL on real java code, do the following:
  1. Download dresden-ocl.jar, royloy.jar and xml4j.jar into a directory.
    Problems when downloading with netscape? Select "Save Link As.." in the context menu, instead of just clicking the link.
  2. Change into that directory.
  3. Unzip the example java code in royloy.jar: 
    jar -xf royloy.jar
  4. Compile the example ocl expressions and put them into the java code: (copy&paste without line breaks) 
    java -cp .:dresden-ocl.jar:xml4j.jar tudresden.ocl.injection.Main \
  --constraint-file  tudresden/ocl/test/royloy/oclexpressions \
  --reflection-model tudresden.ocl.test.royloy \
  --name-adapter      argo \
  --violation-makro  "System.out.println" \
  --modify tudresden/ocl/test/royloy/*.java
    The example java code has now been modified.

    NOTE for JDK1.3 Hotspots VM for Windows NT: The ocl toolkit throws an AbstractMethodError with this VM. Probably this is not NT specific, but specific to Hotspots. Probably this is due to bug #4248990. To fix, recompile the sources with JDK1.3

  5. Compile the example: 
    javac -classpath dresden-ocl.jar:xml4j.jar tudresden/ocl/test/royloy/*.java
  6. Run the example: 
    java -cp .:dresden-ocl.jar:xml4j.jar tudresden.ocl.test.TestInjection
    Enjoy the messages about violated ocl constraints.
  7. Clean the code: (again without line breaks) 
    java -cp .:dresden-ocl.jar:xml4j.jar tudresden.ocl.injection.Main \
  --clean \
  --modify tudresden/ocl/test/royloy/*.java
    The example code is now exactly the same as you downloaded it.
Of course you may try this on your own java code.
Compiler Demo
If you want to use the ocl compiler standalone, do the following:
  1. Download dresden-ocl.jar and xml4j.jar into a directory.
    Problems when downloading with netscape? Select "Save Link As.." in the context menu, instead of just clicking the link.
  2. Change into that directory.
  3. Type 
    java -cp dresden-ocl.jar:xml4j.jar tudresden.ocl.ConstraintEvaluation
    at the command prompt.
  4. A window pops up. There you can enter constraints, check them against a model and generate java code.

NOTE for JDK1.3 Hotspots VM for Windows NT: The ocl toolkit throws an AbstractMethodError with this VM. Probably this is not NT specific, but specific to Hotspots. Probably this is due to bug #4248990. To fix, recompile the sources with JDK1.3

Argo/UML
If you want to use the ocl compiler as part of Argo/UML do the following:
  1. Download ocl-argo.jar (a subset of dresden-ocl.jar).
    Problems when downloading with netscape? Select "Save Link As.." in the context menu, instead of just clicking the link.
  2. Replace the current ocl-argo.jar in your Argo/UML installation with this file.
  3. Restart Argo/UML.
  4. Use the tab "Constraints" in the lower section of the Argo/UML main window.
You do not need xml4j.jar, since this is already part of Argo/UML.
Sources
The complete source including test suites is available in dresden-ocl-src.tar.gz. For compiling, you need xml4j.jar and test.jar. Note, that the ultimate source of the current version is the cvs repository at sourceforge.
Documentation
The javadoc api documentation is available for download or for reading online. You may also generate this from the sources.

...dependencies

These download are offered as a convenience only (and because the compiler is tested with these versions); for more recent versions, visit the original websites.

  • xml4j.jar: XML4J version 2.0.4 by IBM Alphaworks, needed for dresden-ocl.jar, but not for ocl-argo.jar.
  • test.jar: JUnit version 1.0 by Extreme Programming, needed for compiling and running the test suite. The test suite is available in the source package only.
